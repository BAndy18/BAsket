##BAsket|

Help on "Home"

The BAsket is the software complex for a Trading Agent who wants generate Orders for Clients using mobile devices (smartphones and tablets) i.e. to do it quickly, efficiently and with modern technology.

The complex is divided into the Mobile Application (i.e. the BAsket, which you see now) and the Server Module. The Server Module is responsible for the data transmission and storage, as well as "interaction" with the ERP system of the company. The Server Module can be configured to your company's ERP system with plug-ins.

The BAsket can operate in two basic Modes:

  • Offline - data from the server when reading the News is stored into the database (Web Db) of the device. Basically the program works exactly with the local data;
  • Direct Access - all requests are sent directly to the server, and the server's response is displayed directly on your screen.

These two Modes have a lot of advantages and disadvantages, and we give here only a part of them.

The Offline Mode:

  • does not require permanent access to Internet;
  • Internet traffic is very low;
  • system response entirely depends on the speed of your mobile device.

The main advantage of the Direct Access is permanently relevant data. The Stocks-in-Trade as well as the Lists of Goods and Clients on your screen are always as close as possible to your ERP system.

There are additional technical limitations of using the Offline Mode. Devices which use such Internet browsers as Google Chrome (Android), Opera or Safari (iOs), will be able to use the Offline Mode, others will have to be content with the advantages of the Direct Access.

The Program can be used on an ordinary computer, but in this case, the Offline Mode is available only with Google Chrome.

The BAsket is the cross-platform and cross-browser application that is written with Single-Page Application modern technology. Such technologies as HTML5, CSS3, JS-frameworks PhoneJS, JQuery, Knockout are used here. In fact - the BAsket is the website that can also work as a native application for mobile devices, thanks to PhoneGap technology.

It is natural that this application works slower then native application on a mobile device. But, first, device productivity is growing very fast, and secondly, the ability to work with one software on different types of devices promises great benefits from using.

Although the program is announced to support communicators, we still advise using tablets for real work, because screen size is very importan here.

The BAsket contains the following functionality:

  • Preparation and correction of the Client Orders;
  • Working with the List of Orders;
  • The List of Road Maps, which determines the Route of a Trading Agent;
  • The List of Clients, where you can see the Route from the current location to a particular Trading Point;
  • The News Screen, where information is exchanged with the server;
  • The System Configuration, where among others you can "play" with the interface styles of various types of mobile devices.
##Order|

Help on "Order"

Preparation and correction of the Order is made on three linked screens:

  • The Order
  • The Good Selection
  • The Goods Quantity

On this screen you have to specify the "headline" parameters of the Order:

  • Date - the date of considered shipment of the Order;
  • Client - Selection of the Client, to whom the Oder will be shiped;
  • Trading Point - Selection of the Client's Trading Point, where the Order will be shiped. If the selected Client has no Trading Points, this field is hidden;
  • * The Company - selection of the corporate body for which the Order will be made;
  • * Payment Type - selection of payment type for the Order;
  • Notes - any notes to the Order;
  • Information about how many Goods are selected into the Order and for what Sum.
* parameters are determined by the configuration of the program and may be omitted.

By clicking  the "Save" button the validation of the Order is checked and if everything is done correctly, the Order is saved in the List of Orders. Then transition to the screen from which you came here takes place. It may be the Home, the List of Orders or the Road Maps.

If the check failed, the error message pops up. Here is the list of error situations:

  • Goods were not selected;
  • Date was not set;
  • Client was not selected.

The "Save" button may be omitted if the Order is corrected in the "Read Only" mode.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the previous screen takes place;
  • The Screen Name;
  • The "I" button - brings up this Help screen;
  • The "Goods" button - the transition to the Good Selection screen takes place.
##Products|

Help on "Good Selection"

The "Good selection" is the second of three linked screens for the Order preparation:

  • The Order
  • The Good Selection
  • The Goods Quantity

This screen is used for the Good Selection into the Order. The screen comes in two modes: the "Good Selection" and the "Basket".

1) In the "Good Selection" you can select Goods into the Order by clicking on the Good Name in the List, change the Category of Goods and filter Goods within the Selected Category.

On the left of the line above the List of Goods there is a window for the List of Goods of selected Category searching (filtering). On the right there is the Good Category selection button. When you click on it the List of Category Selection of Goods pops up.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the Order screen takes place;
  • The Screen Name;
  • The "I" button - brings up this Help screen;
  • The "Basket" button - changes the screen mode on the "Basket".

2) In the "Basket" you can look through and correct the List of Goods selected into the Order.

The line above the List of Goods contains information about how many Goods are selected into the Order and for what Sum.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the Order screen takes place;
  • The Screen Name;
  • The "I" button - brings up this Help screen;
  • The "Goods" button - changes the screen mode on the "Good Selection".

In both Modes when clicking on a particular Good you get to the next screen - the Goods Quantity - to set the number of the Goods selected into the Order.

Each List item contains:

  • The 1st line - [Goods Quantity in the Order] (is omitted if Good is not selected into the Order)
  • The 2nd line - [Good Name]
  • The 3rd line - [Good Price] ([Notes])

##OrderList|

Help on "List of Orders"

The List of Orders is displayed in two modes:

  • The Local Orders - the List of Orders that is stored in the local database of the device. To look through this List Internet connection is not required;
  • The Server Orders - the List of Orders that is stored on the server. To look through this List Internet connection is required.

Switching between modes is made by clicking on the button with the same changing text in the upper right corner of the screen.

On the left in the same line there is the box for searching (filtering) the List of Orders by the Name of the Client.

By clicking on the line of the List of Orders the transition to the Order screen takes place. Thus it is possible to correct the Order both in the local database and on the server. It depends on the selected mode.

By a long clicking (about 0.5 seconds) on the line of the List the Context Menu pops up in the bottom of the screen. It contains the following items:

  • Delete the Order - the Order is deleted according to the mode, either locally or on the server. To complete the deletion the confirmation is required;
  • Change Activity - action is enabled only in the "Local Orders" mode. It switches the indicator of the necessity of sending the Selected Order to the server from the News screen. Inactive Orders (that were already sent to the server) are placed at the bottom of the List and are marked with grey background;
  • Cancel - closes the Context Menu.

Each List item contains:

  • The 1st line - [Order Number] - [Short Date] [Client Name] - [Trading Point Name (if any)] [Shipping Address]
  • The 2nd line - [Order Sum] ([Notes])

In the title bar of the screen there are (from the left to the right):

  • The "Menu" button - brings up the BAsket Sidebar Menu;
  • The Screen Name;
  • The "I" button - brings up this Help screen.

Help on "Client"

Here the current point (your position) and the point of the Client Address are indicated on the Map. You have to understand that if the Client Address was not stored properly in the system, the point on the Map can be indicated in the most unexpected place. :)

Correct Address contains the following information separated by commas:

  • House number and Street name;
  • City name;
  • Country (may be usually omitted);
  • Postcode.

The Name and the Address of the Client are indicated in the line above the Map.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the List of Clients screen takes place;
  • The Screen Name;
  • The "I" button - brings up this Help screen.
  • The "=" button - brings up the Menu.

The Menu contains following items:

  • Save - nothing carries out yet :)
  • The Screen Name;
  • Detailed Route - you can see the window with following information: the Google Map text information about the distance to the Trading Point, travel time (by car) and detailed information about moving to the Client through the Points of requirement, similar to information from the car navigator.
##RoadMapList|

Help on "Road Maps" on the date

Here you can see the List of Clients included in the Road Map on the set date. In this way the Client to Client Route on a particular day of Trading Agent is shown.

In the line above the List of Clients there are (from the left to the right):

  • The Date - changes the List of Road Maps on the date;
  • The "Add Client" button - brings up the dialog for finding and adding a new Client to the Road Maps on the date;

By clicking on the line of the List of Clients the transition to the Order screen takes place. If the Order for the current Client has not yet been created, it will be created automatically, otherwise the Order opens in the Correction Mode.

By a long clicking (about 0.5 seconds) on the line of the List the Context Menu pops up in the bottom of the screen. It contains the following items:

  • Open Client - the transition to the Client screen to see Client on the Map takes place;

  • Move Down - the current Client moves down in the List;
  • Delete Client - the current Client is removed from the Road Map;
  • Cancel - closes the Context Menu.

Each List item contains:

  • The 1st line - [Sequence number of the Client in the Route] [Order number (if already created)] [Client Name - Trading Point]
  • The 2nd line - ([Shipping Address])

In the title bar of the screen there are (from the left to the right):

  • The "Menu" button - brings up the BAsket Sidebar Menu;

##ReadNews|

Help on "News"

Here you can send the prepared Order to the server and update the Directories in the Offline Mode.

If the operation without using Web Db is selected in the Preferences screen the message "No data" pops up.

Otherwise, you can set the value of the following switches:

  • Send Orders - if "ON" the prepared Orders that are in the active state go to the server;
  • Update Stocks-in-Trade - if "ON" the Stocks-in-Trade only will be read from the server;
  • Update Directories and Maps - if "ON" all updates of the Goods and Clients Directories as well as new Road Maps (from the current date and further) will be read from the server.

By clicking on the "Load Data" button the process of data exchanging with the server will start.

In the title bar of the screen there are (from the left to the right):

  • The "Menu" button - brings up the BAsket Sidebar Menu;
  • The Screen Name;
  • The "I" button - brings up this Help screen.

Help on "Preferences"
