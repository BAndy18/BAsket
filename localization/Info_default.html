##Home|

Help on "Home"

BAsket - a software complex for trading agent who wants generate Orders for Clients using mobile devices (smartphones and tablets) i.e. to do it quickly, efficiently and with modern technology.

The complex is divided into the Mobile Application (i.e. BAsket, which you see now) and the Server Module. The Server Module is responsible for the data transmission and storage, as well as "interaction" with the ERP system of the company. The Server module can be configured to your company's ERP system with plug-ins.

BAsket can operate in two basic modes:

  • Offline - data from the server when reading the News is stored into database (Web Db) of the device. Basically the program works exactly with the local data;
  • Direct Access - all requests are sent directly to the server, and the server's response is displayed directly on your screen.

These two modes have a lot of advantages and disadvantages, and we give here only a part of them.

Offline Mode:

  • does not require permanent access to the Internet;
  • Internet traffic is very low;
  • system response entirely depends on the speed of your mobile device.

The main advantage of Direct Access is permanently relevant data. Stocks-in-Trade as well as the Lists of Goods and Clients on your screen are always as close as possible to your ERP system.

There are additional technical limitations of using Offline Mode. Devices which use such an Internet browsers as Google Chrome (Android), Opera or Safari (iOs), will be able to use Offline Mode, others will have to be content with the advantages of Direct Access.

Program can be used on an ordinary computer, but in this case, the Offline Mode is available only with Google Chrome.

BAsket - the cross-platform and cross-browser application that is written with Single-Page Application modern technology. Such technologies as HTML5, CSS3, JS-frameworks PhoneJS, JQuery, Knockout are used here. In fact - BAsket is the website that can also work as a native application for mobile devices, thanks to PhoneGap technology.

It is natural that this application works slower then native application on a mobile device. But, first, device productivity is growing very fast, and secondly, the ability to work with one software on different types of devices promises great benefits from using.

Although the program is announced to support communicators, we still advise using tablets for real work, because screen size is very importan here.

BAsket contains the following functionality:

  • ,
  • Preparation and correction of Client Orders
  • Working with a List of Orders;
  • List of Road Maps, which determines the route of trading agent;
  • List of Clients, where you can see the route from the current location to a particular trading point;
  • News Screen, where information is exchanged with the server;
  • System Configuration, where among others you can "play" with the interface styles of various types of mobile devices.
##Order|

Help on "Order"

Forming and correction of the Order is made on three screens connected with links:

  • Order
  • Good Selection
  • Good Quantity

On this screen you have to specify the "headline" parameters of the Order:

  • Date - the date of considered shipment of the Order;
  • Client - selection of a Client, to whom the Oder will be shiped;
  • Trading Point - Selection of the Client's Trading Point, where the Oder will be shiped. If the selected Client has no Trading Points, this field is hidden;
  • * The Company - selection of the corporate body for which the Order will be issued;
  • * Payment Type - selection of payment type for the Order;
  • Notes - any notes to the Order;
  • Information about how many goods are selected in the Order and for what sum.
* parameters are determined by the configuration of the program and may be missing.

By clicking  the "Save" button the validity of the Order is checked and if everything is done correctly, the Order is saved in Orders List. Then transition to the screen from which you came here is made. It may be Home, Orders List or Road Maps.

Если проверка завершилась с ошибкой, внизу всплывает сообщение об ошибке. Список ошибочных ситуаций:

  • Не выбраны товары в Заказе;
  • Не установлена дата в Заказе;
  • Не выбран клиент в Заказе.

Кнопка "Сохранить" может отсутствовать, если Заказ корректируется в состоянии "Только для чтения".

В титульной строке на экране слева направо находятся:

  • Кнопка "назад" - переход к предыдущему экрану;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки;
  • Кнопка "Товары" - переход на экран Выбор товаров.
##Products|

Help on "Goods Selection"

Выбор товаров - это второй из трех, связанных переходами, экранов для формирования Заказа:

  • Order
  • Выбор товаров
  • Количество товара

Этот экран предназначен для выбора товаров в Заказ. Экран существует в двух режимах: "Выбор Товаров" и "Корзина".

1) В режиме "Выбор Товаров" можно выбать товар в Заказ, нажав на элемент списка, сменить категорию Товаров и фильтровать Товары в рамках выбранной категории.

В строке над списком Товаров слева есть окно для поиска (фильтрации) списка Товаров выбранной категории. Справа находится кнопка выбора категории Товара. При ее нажатии всплывает список выбора категорий Товара.

В титульной строке слева направо находятся:

  • Кнопка "назад" - переход к экрану Заказ;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки;
  • Кнопка "Корзина" - меняет режим экрана на "Корзина".

2) В режиме "Корзина" можно просматривать и корректировать список выбранных в Заказ Товаров.

Строка над списком Товаров содержит информацию о том, сколько выбрано товаров в Заказ и на какую сумму.

В титульной строке слева направо находятся:

  • Кнопка "назад" - переход к экрану Заказ;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки;
  • Кнопка "Товары" - меняет режим экрана на "Выбор товаров".

В обоих режимах при нажатии на конкретный товар мы попадаем на следующий экран - Количество товара - для указания количества выбранного Товара в Заказ.

Каждый элемент списка содержит:

  • 1-я строка - [Количество в Заказе] (отсутствует, если Товар не выбран в Заказ)
  • 2-я строка - [Наименование Товара]
  • 3-я строка - [Цена Товара] ([Примечания])

##OrderList|

Help on "Список Заказов"

Здесь показывается Список Заказов в двух режимах:

  • Локальные Заказы - это заказы, сохраненные в локальной базе данных устройства. Для просмотра этого списка не требуется подключение к сети Интернет;
  • Заказы на сервере - это список сохраненных на сервере Заказов. Для просмотра требуется подключение к Интернет.

Переключение между режимами производится кнопкой в правом верхнем углу экрана с одноименным меняющимся текстом.

В этой же строке слева есть поле для поиска (фильтрации списка) Заказов по имени Клиента.

При нажатии на строку списка Заказов происходит переход на экран Заказ. При этом можно корректировать Заказ как в локальной базе данных, так и на сервере. Это зависит от выбранного режима.

При длительном (примерно 0.5 секунды) нажатии на строку списка снизу выпадает контекстное меню со следующими пунктами:

  • Удалить Заказ - Заказ удаляется в зависимости от режима, локально или на сервере. Для выполнения действия требуется подтвердить операцию удаления;
  • Сменить активность - Действие активно только в режиме "Локальные Заказы" и переключает признак необходимости отправки выбранного Заказа на сервер из экрана Новости. Неактивные (уже отправленные на сервер) Заказы располагаются внизу списка и отмечаются серым фоном;
  • Отменить - закрывает контекстное меню.

Каждый элемент списка содержит:

  • 1-я строка - [Номер Заказа] - [Дата сокращенно] [Имя клиента] - [Имя точки (если есть)] [Адрес Доставки]
  • 2-я строка - [Сумма Заказа] ([Примечания])

В титульной строке на экране слева направо находятся:

  • Кнопка "меню" - вызов бокового меню BAsket;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки.

##Client|

Help on "Client"

Здесь на карте показывается текущая точка (Ваше положение) и точка адреса Клиента. Надо иметь в виду, что если адрес Клиента в системе сохранен неправильно, то и точка на карте может быть отмечена в самом неожиданном месте :)

Правильным адресом считается наличие следующей информации через запятую:

  • Дом и улица;
  • Город;
  • Страна (обычно можно опустить);
  • Почтовый индекс.

В строке над картой указывается имя и адрес Клиента.

В титульной строке на экране слева направо находятся:

  • Кнопка "назад" - возврат к экрану Список Клиентов;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки;
  • Кнопка "=" - вызывает меню.

Меню содержит пункты:

  • Сохранить - ничего пока не делает :)
  • Название экрана;
  • Маршрут подробно - выводится окно с текстовой информацией от Google Map о расстоянии до точки Клиента, времени прохождения (на автомобиле) и подробную информацию по пунктам предписания о передвижении до Клиента, аналогично информации с автомобильного навигатора.
##RoadMapList|

Help on "Road Maps" on the date

Здесь показывается список Клиентов, включенных в Дорожную карту на установленную дату. Этим самым показывается маршрут движения торгового агента по Клиентам в определенный день.

В строке над списком Клиентов слева направо:

  • Дата - меняет список Дорожной карты на дату;
  • Кнопка "Добавить клиента" - вызывает диалог для поиска и добавления нового клиента в Дорожную карту на дату;
  • Кнопка "Маршрут на карте" - переход на экран Дорожная Карта.

При нажатии на строку списка Клиентов происходит переход на экран Заказ. Если Заказ для текущего Клиента еще не создан, он создается, иначе Заказ открывается в режиме корректировки.

При длительном (примерно 0.5 секунды) нажатии на строку списка снизу выпадает контекстное меню со следующими пунктами:

  • Открыть Клиента - переход на экран Клиент для просмотра Клиента на карте;
  • Переместить вверх - текущий Клиент перемещается вверх по списку, корректируя маршрут движения по Дорожной карте;
  • Переместить вниз - текущий Клиент перемещается вниз по списку;
  • Удалить Клиента - текущий Клиент удаляется из Дорожной катры;
  • Отменить - закрывает контекстное меню.

Каждый элемент списка содержит:

  • 1-я строка - [Порядковый номер Клиента в маршруте] [Номер Заказа (если уже создан)] [Имя Клиента - Точка]
  • 2-я строка - ([Адрес доставки])

В титульной строке на экране слева направо находятся:

  • Кнопка "меню" - вызов бокового меню BAsket;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки.

##ReadNews|

Help on "News"

Здесь в режиме автономной работы можно отсылать сформированные Заказы на сервер и обновлять справочники.

Если в экране Настройка выбрана работа без использования Web Db, будет показано сообщение "Нет данных".

Иначе можно установить значение следующих переключателей:

  • Отправить Заказы - если "ON", на сервер отправятся подготовленные Заказы, находящиеся в активном состоянии;
  • Обновить товарные остатки - если "ON", с сервера прочитаются только остатки по Товарам;
  • Обновить справочники и карты - если "ON", с сервера прочитаются все обновления по справочникам Товаров и Клиентов, а также новые (начиная с текущей даты и позднее) Дорожные карты.
При нажатии на кнопку "Загрузка Данных" начнется процесс обмена данными с сервером.

В титульной строке на экране слева направо находятся:

  • Кнопка "меню" - вызов бокового меню BAsket;
  • Название экрана;
  • Кнопка "I" - вызывает этот экран справки.

Help on "Preferences"

Здесь можно произвести некоторую настройку программы. Настройка имеет следующую структуру:

  • Основные свойства:

    • Стиль Интерфейса - вызывает список тем, которые можно использовать как основной стиль;

    • Табличный выбор товаров - переключатель режима работы на экране Выбор товаров.
  • Пользователь:

    • Имя пользователя - имя, по которому сервер будет "узнавать" пользователя;
    • Пароль на сервере - надо устанавливать, если работа ведется не с мобильного устройства;
    • EMail - адрес EMail пользователя (опционально).
  • Админ - некоторые административные настройки программы, защищенные паролем
