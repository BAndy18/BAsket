##BAsket|

Help on "Home"

BAsket - a software complex for a trading agent who wants generate Orders for Clients using mobile devices (smartphones and tablets) i.e. to do it quickly, efficiently and with modern technology.

The complex is divided into the Mobile Application (i.e. BAsket, which you see now) and the Server Module. The Server Module is responsible for the data transmission and storage, as well as "interaction" with the ERP system of the company. The Server Module can be configured to your company's ERP system with plug-ins.

BAsket can operate in two basic modes:

  • Offline - data from the server when reading the News is stored into database (Web Db) of the device. Basically the program works exactly with the local data;
  • Direct Access - all requests are sent directly to the server, and the server's response is displayed directly on your screen.

These two modes have a lot of advantages and disadvantages, and we give here only a part of them.

Offline Mode:

  • does not require permanent access to the Internet;
  • Internet traffic is very low;
  • system response entirely depends on the speed of your mobile device.

The main advantage of Direct Access is permanently relevant data. Stocks-in-Trade as well as the Lists of Goods and Clients on your screen are always as close as possible to your ERP system.

There are additional technical limitations of using Offline Mode. Devices which use such an Internet browsers as Google Chrome (Android), Opera or Safari (iOs), will be able to use Offline Mode, others will have to be content with the advantages of Direct Access.

Program can be used on an ordinary computer, but in this case, the Offline Mode is available only with Google Chrome.

BAsket - the cross-platform and cross-browser application that is written with Single-Page Application modern technology. Such technologies as HTML5, CSS3, JS-frameworks PhoneJS, JQuery, Knockout are used here. In fact - BAsket is the website that can also work as a native application for mobile devices, thanks to PhoneGap technology.

It is natural that this application works slower then native application on a mobile device. But, first, device productivity is growing very fast, and secondly, the ability to work with one software on different types of devices promises great benefits from using.

Although the program is announced to support communicators, we still advise using tablets for real work, because screen size is very importan here.

BAsket contains the following functionality:

  • ,
  • Preparation and correction of the Client Orders
  • Working with the List of Orders;
  • The List of Road Maps, which determines the route of a trading agent;
  • The List of Clients, where you can see the route from the current location to a particular trading point;
  • The News Screen, where information is exchanged with the server;
  • The System Configuration, where among others you can "play" with the interface styles of various types of mobile devices.
##Order|

Help on "Order"

Forming and correction of the Order is made on three linked screens:

  • The Order
  • The Good Selection
  • The Goods Quantity

On this screen you have to specify the "headline" parameters of the Order:

  • Date - the date of considered shipment of the Order;
  • Client - Selection of a Client, to whom the Oder will be shiped;
  • Trading Point - Selection of the Client's Trading Point, where the Order will be shiped. If the selected Client has no Trading Points, this field is hidden;
  • * The Company - selection of the corporate body for which the Order will be made;
  • * Payment Type - selection of payment type for the Order;
  • Notes - any notes to the Order;
  • Information about how many Goods are selected into the Order and for what Sum.
* parameters are determined by the configuration of the program and may be missed.

By clicking  the "Save" button the validity of the Order is checked and if everything is done correctly, the Order is saved in the List of Orders. Then transition to the screen from which you came here is made. It may be the Home, the List of Orders or the Road Maps.

If the check failed, the error message pops up. Here is the list of error situations:

  • Goods were not selected;
  • Date was not set;
  • Client was not selected.

The "Save" button may be missed if the Order is corrected in the "Read Only" mode.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the previous screen;
  • Screen Name;
  • The "I" button - brings this Help screen;
  • The "Goods" button - the transition to the Good Selection screen.
##Products|

Help on "Good Selection"

"Good selection" is the second of three linked screens for the Order formation:

  • The Order
  • The Good Selection
  • The Goods Quantity

This screen is used for Good Selection into the Order. The screen comes in two modes: "Good Selection" and "Basket"

1) In the "Good Selection" you can select Goods into the Order by clicking on the Good name in the list, change the Category of Goods and filter Goods within the Selected Category.

On the left of the line above the List of Goods there is a window for the List of Goods of selected Category searching (filtering). On the right there is the Good Category selection button. When you click on it the List of Category Selection of Goods pops up.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the Order screen;
  • Screen Name;
  • The "I" button - brings this Help screen;
  • The "Basket" button - change the screen mode on "Basket".

2) In the "Basket" you can look through and correct the List of Goods selected into the Order.

The line above the List of Goods contains information about how many goods are selected into the Order and for what sum.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the Order screen;
  • Screen Name;
  • The "I" button - brings this Help screen;
  • The "Goods" button - change the screen mode on "Good Selection".

In both modes when clicking on a particular Good we get to the next screen - the Goods Quantity - to set the number of the Goods selected into the Order.

Each List item contains:

  • The 1st line - [Goods Quantity in the Order] (missing if Good is not selected into the Order)
  • The 2nd line - [Good Name]
  • The 3rd line - [Good Price] ([Notes])

##OrderList|

Help on "List of Orders"

Here the List of Orders is displayed in two modes:

  • The Local Orders - the Orders that are stored in the local database of the device. To look through this list an Internet connection is not required;
  • The Server Orders - the List of Orders that is saved on the server. To look through this List Internet connection is required.

Switching between modes is made by clicking on the button with the same changing text in the upper right corner of the screen.

On the left in the same line there is the box for searching (filtering) the List of Orders by the name of the Client.

By clicking on the line of the List of Orders the transition to the Order screen takes place. Thus it is possible to correct the Order both in the local database and on the server. It depends on the selected mode.

By a long clicking (about 0.5 seconds) on the line of the List the context menu appears in the bottom of the screen. It contains the following items:

  • Delete the Order - the Order is deleted according to the mode, either locally or on the server. To complete the deletion the confirmation is required;
  • Change Activity - action is enabled only in the "Local Orders" mode. It switches the indicator of the necessity of sending the Selected Order to the server from the News screen. Inactive Orders (that were already sent to the server) are placed at the bottom of the list and are marked with gray background;
  • Cancel - closes the context menu.

Each List item contains:

  • The 1st line - [the Order Number] - [the Short Date] [the Client Name] - [the Trading Point Name (if any)] [the Shipping Address]
  • The 2nd line - [the Order Sum] ([Notes])

In the title bar of the screen there are (from the left to the right):

  • The "menu" button - brings the BAsket sidebar menu;
  • Screen Name;
  • The "I" button - brings this Help screen.

##Client|

Help on "Client"

Here the current point (your position) and the point of the Client Address indicate on the map. We have to understand that if the Client Address was not stored properly in the system, the point on the map can be indicated in the most unexpected place. :)

Correct address contains the following information separated by commas:

  • House number and Street name;
  • City name;
  • Country (can be usually omitted);
  • Postcode.

The Name and the Address of the Client indicate in the line above the map.

In the title bar of the screen there are (from the left to the right):

  • The "Back" button - the transition to the List of Clients screen;
  • Screen Name;
  • The "I" button - brings this Help screen.
  • The "=" button - brings the Menu.

Menu contains following items:

  • Save - nothing carries out yet :)
  • The Screen Name;
  • Detailed route - we can see the window with following information: the Google Map text information about the distance to the Trading Point, travel time (by car) and detailed information about moving to the Client through the points of requirement, similar to information from the car navigator.
##RoadMapList|

Help on "Road Maps" on the date

Here we can see the list of Clients included in the Road Map on the set date. In this way the Client to Client Route on a particular day of Trading Agent is shown.

In the line above the List of Clients there are (from the left to the right):

  • Date - changes the List of Road Maps on the date;
  • The "Add Client" button - brings up the dialog for finding and adding a new Client to the Road Maps on the date;

By clicking on the line of the List of Clients the transition to the Order screen takes place. If the Order for current client has not yet been created, it is created, otherwise the Order opens in the Correction Mode.

By a long clicking (about 0.5 seconds) on the line of the List the context menu appears in the bottom of the screen. It contains the following items:

  • Open Client - the transition to the Client screen to see Client on the Map;
  • Move Up - current Client moves up in the List with correction of the Route on the Road Map;
  • Move Down - current Client moves down in the List;
  • Delete Client - current Client removed from the Road Map;
  • Cancel - closes the context Menu.

Each List item contains:

  • The 1st line - [Sequence number of the Client in the Route] [Order number (if already created)] [Client Name - Trading Point]
  • The 2nd line - ([the Shipping Address])

In the title bar of the screen there are (from the left to the right):

  • The "menu" button - brings the BAsket sidebar menu;
  • Screen Name;
  • The "I" button - brings this Help screen.

##ReadNews|

Help on "News"

Here we can send the prepared Order to the server and Update the Directories in the Offline Mode.

If the operation without using Web Db is selected in the Preferences screen the message "No data" pops up.

Otherwise, you can set the value of the following switches:

  • Send Orders - if "ON" the prepared Orders that are in the active state go to the server;
  • Update Stocks-in-Trade - if "ON" the Stocks-in-Trade only will be read from the server;
  • Update Directories and Maps - if "ON" all updates of the Goods and Clients Directories as well as new Road Maps (from the current date and further) will be read from the server.

By clicking on the "Load Data" button the process of data exchanging with the server will start.

In the title bar of the screen there are (from the left to the right):

  • The "menu" button - brings the BAsket sidebar menu;
  • Screen Name;
  • The "I" button - brings this Help screen.

Help on "Preferences"

Here you can make some program Configuration. Configuration has the following structure:

    • User Name - the Name by which the server can "recognize" the User;
    • Server User Password - should be installed if you are not on a mobile device;
    • EMail - User EMail (optional).
  • Admin - some administrative password-protected program Configurations.
