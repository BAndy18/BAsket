BAsket - это программный комплекс для торгового агента, который хочет формировать Заказы для Клиентов с использованием мобильных устройств (планшетов и коммуникаторов), то есть делать это быстро, эффективно и современно.

Комплекс делится на Мобильное приложение и Серверный модуль, отвечающий за передачу и хранение данных, а также за "общение" с учетной системой. Серверный модуль может быть настроен на учетную систему Вашего предприятия при помощи плагинов.

2)

BAsket может работать в двух основных режимах:

  • Автономно - данные с сервера при чтении Новостей запоминаются в базе данных (Web Db) устройства. В основном программа работает именно с локальными данными;
  • Прямой доступ - все запросы поступают непосредственно на сервер, и ответ сервера отображается сразу на Вашем экране.

BAsket - это кроссплатформенное и кроссбраузерное приложение, написанное с использованием современной технологии Single-Page Application. Здесь использованы такие технологии, как HTML5, CSS3, JS-фреймворки PhoneJS, JQuery, Knockout. На самом деле - это сайт, который может работать еще и как нативное приложение для мобильных устройств, благодаря технологии PhoneGap.

Естественно, что такое приложение на мобильном устройстве будет работать медленнее нативного. Но, во-первых, производительность устройств растет очень быстро, а во-вторых, возможность работать с одним продуктом на разных типах устройств сулит большую выгоду от использования.

Хоть программа и анонсирована на поддержку коммуникаторов, мы все же советуем для реальной работы использовать планшеты. Размер экрана здесь имеет очень большое значение.

Функционально BAsket содержит:

  • Подготовку и корректировку Заказов для Клиентов;
  • Работу со Списком Заказов;
  • Список Дорожные карты, определяющий маршрут движения торгового агента;
  • Список Клиентов, где можно посмотреть маршрут от текущего положения до конкретной торговой точки;
  • Экран Новости, где производится обмен информацией с сервером;
  • Настройку системы

Общие сведения

BAsket - это программный комплекс для торгового агента, который хочет формировать Заказы для Клиентов с использованием мобильных устройств (планшетов и коммуникаторов), то есть делать это быстро, эффективно и современно.

Комплекс делится на Мобильное приложение и Серверный модуль, отвечающий за передачу и хранение данных, а также за "общение" с учетной системой. Серверный модуль может быть настроен на учетную систему Вашего предприятия при помощи плагинов.

Достоинства и недостатки этих двух режимов многочисленны, и мы приведем здесь только их часть.

Автономный режим:

  • не требует постоянного доступа к Интернет;
  • значительно "дешевле" по интернет трафику;
  • скорость ответа системы уже полностью зависит от скорости работы Вашего мобильного устройства.

Основное достоинство Прямого доступа - постоянно актуальные данные. Товарные остатки, а также списки Товаров и Клиентов на Вашем экране постоянно находятся в состоянии, максимально приближенном к Вашей учетной системе.

Есть еще технические ограничения на использование Автономного режима. Устройства, которые используют в качестве Интернет браузера Google Chrome (Android), Opera или Safari (iOs), смогут пользоваться Автономным режимом, остальным придется довольствоваться достоинствами Прямого доступа.

Программой можно пользоваться и с обычного компьютера, но и в этом случае Автономный режим доступен только из Google Chrome.

Здесь использованы такие технологии, как HTML5, CSS3, JS-фреймворки PhoneJS, JQuery, Knockout. ##Server|

Серверный модуль

Выполнен по технологии ASP.NET Web Api. Требует развертывания и настройки IIS интернет-сервера, .NET 4.0 и MS Sql Server версии 2005 и выше

Модуль может обновлять информацию по справочникам из XML файла, который может быть подготовлен в Вашей учетной системе. Подготовленные Заказы также выгружаются в формате XML.

Более подробная информация по настройке Серверного модуля находится в файле readme в архиве ws.zip
